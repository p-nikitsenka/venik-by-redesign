## Стек и деплой

Статика без сборки: `index.html` + `styles.css` + `app.js` + `assets/`.
GitHub Pages из ветки `main`, домен borshevik.by (файл `CNAME` в корне).
Локальный клон: `C:\venik-by-redesign\venik-by-redesign` (внимание: вложенная папка, внешняя — не репозиторий).

**Пуш = деплой.** Через 1–2 минуты после push сайт обновляется.

## Правила работы

1. **Коммит и push делает владелец сам.** Задача Claude Code — внести правки, показать `git diff` и остановиться. Не коммитить без явной просьбы.
2. **`CNAME` не трогать никогда.** Его удаление отвяжет домен.
3. **CSS-правила заменять, а не дублировать.** Частая ошибка: добавить новое правило, оставив старое. Каскад применит последнее, но файл захламляется и путает при следующих правках. Перед добавлением — проверить, нет ли уже такого селектора.
4. **Проверять результат счётчиком, а не глазами.** `(Select-String -Path index.html -Pattern '...' -AllMatches).Matches.Count` — надёжнее чтения диффа в терминале, который переносит длинные строки и выглядит как задвоение.
5. **Файлы `*_big.*` игнорируются** (см. `.gitignore`) — это локальные резервные копии тяжёлых исходников.

## Дизайн-система

### Палитра (переменные в `:root`)

| Токен | Значение | Где |
|---|---|---|
| `--bg` | `#f5f6f1` | основной фон страницы |
| `--soft` | `#e4e8dc` | чередующиеся светлые секции |
| `--surface` | `#fff` | карточки, белые секции |
| `--ink` | `#111310` | основной текст |
| `--muted` | `#62675f` | вторичный текст (5.33:1 на `--bg` — проходит) |
| `--line` | `#dde0d8` | границы |
| `--lime` | `#b8d700` | акцент: фон кнопок, плашки, полосы. С чёрным текстом 11.47:1 |
| `--lime-dark` | `#5c6a00` | лаймовый **текст** на светлом фоне (white 5.97:1, `--bg` 5.5:1, `--soft` 4.8:1 — проходит везде) |
| `--graphite` | `#151714` | тёмные секции, primary-кнопки |

**Важно про `--lime-dark`:** был `#91a900` (2.45:1 — не проходил даже для крупного текста), затем `#657600` (4.07:1 на `--soft` — не проходил для мелкого). Любой лаймовый текст на светлом фоне — только через `var(--lime-dark)`, без хардкода. Нельзя использовать `--lime` (#b8d700) как цвет текста на светлом фоне — он для заливок.

### Типографика

Шрифты: **Golos Text** (400–900) — основной, **IBM Plex Mono** (500–700) — цифры, технические подписи, «ВДГ» в логотипе.

| Элемент | Размер | Вес |
|---|---|---|
| `h1` | `clamp(42px,6vw,70px)`, мобильный 30px | **600** |
| `h2` | `clamp(30px,4.3vw,46px)` | **600** |
| `.eyebrow`, `.label` | 12px, uppercase, `letter-spacing:.11em` | **700** |
| `.btn` | 18px, `padding:15px 24px` | **700** |
| `.brandname` (логотип) | 34px / 28px мобильный | **900** |
| body | 16px | 400 |

**Принцип:** вес 900 — только у логотипа. Раньше почти всё было 900, страница выглядела «толстой»: когда жирное везде, оно перестаёт быть акцентом. Заголовки в 600 при крупном кегле читаются легче.

### Логотип

Шапка: лаймовая вертикальная черта + «ВЕНИК» (Golos Text 900) + «ВДГ» (IBM Plex Mono, надстрочно, `--muted`). Отдельного знака у продукта нет — не выдумывать.
Футер: `assets/logo-kp-final.svg` — логотип «Китайские препараты» (белый текст + красный знак, прозрачный фон), по центру над реквизитами.

### Брейкпоинты

`@media(max-width:980px)` → `@media(max-width:720px)` → `@media(max-width:380px)`

**Грабли:** блок 380px идёт последним и перебивает 720px. Если правило есть в обоих — победит 380. Проверять при любой правке размеров.

## Доступность (обязательно)

- Контраст **4.5:1** для текста до 24px, **3:1** для крупного. Проверять любой новый цвет.
- Touch-target минимум **44×44px** (`.iconbtn` — 44px).
- `alt` — описательный для смысловых картинок, `alt=""` для декоративных (иконки аудитории).
- `aria-label` для кнопок без текста (бургер, звонок, Telegram).
- Мелкий текст не меньше 12px.

## Структура страницы

hero → «Для кого» (`.white`) → «Проблема» (`.soft`) → «Системное решение» (`#application`, тёмный блок) → «Регламент» (`#registry`, `.soft`) → «Результат» (`#results`, `.white`, фото до/после + тайм-лапс) → «Калькулятор» (`#calculator`, `.calc`, тёмный) → «Как купить» (`#buy`) → FAQ (`#faq`) → `.final` (лаймовый) → футер.

Тёмных якорей два: «Системное решение» и «Калькулятор». Тёмная тема **не основная** — светлая. Не добавлять тёмного без необходимости: от тёмно-зелёного фона ушли сознательно.

## Бэкенд заявок

Форма → `POST https://api.borshevik.by/lead` (FastAPI на Contabo, systemd `venik-leads`, порт 8090 за nginx).
Ответ: `{"ok": true, "tg": "https://t.me/venik_leads_bot?start=lead_XXXX"}` — ссылка на бота с привязкой к заявке.
Защита: rate-limit 5/10мин по IP, honeypot-поле `website` (скрытое, боты заполняют — заявка молча отбрасывается).
Заявки падают в Telegram-группу «Веник — заявки». Диалоги клиент↔менеджер — бот `venik-dialog` (aiogram), ответ реплаем.

Фронт: при ошибке сети — запасной путь, копирование заявки в буфер обмена.
